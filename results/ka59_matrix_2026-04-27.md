## Headline finding

**Win-rate cannot discriminate axes** — every cell is 0%. **Click count discriminates them, and only for the reasoning model.**

- **gpt-4.1 click count is flat (~90 across all configs).** The non-reasoning model click-spams identically regardless of which axis is hardened. It treats the game as a button to mash.
- **gpt-5.2 click count varies by axis (0 → 94).** The reasoning model adjusts behavior based on what's been knocked out — *except* when mechanics is hardened, where it collapses back to gpt-4.1-level click-spamming.

| Behavior signature | gpt-5.2 | Interpretation |
|---|---:|---|
| `world_hard` → 2 clicks | -94% vs baseline | Refuses to act under degraded observation |
| `feedback_hard` → 23 clicks | -28% vs baseline | Reduced engagement when feedback hidden |
| `baseline` / `goal_hard` → 32–38 | reference | Moderate click rate |
| `mechanics_hard` → 94 clicks | +194% vs baseline | **Reasoning advantage collapses** |

## Methodological note for the paper

The `mechanics_hard` axis specifically **defeats the reasoning advantage** — gpt-5.2's behavioral signature converges with gpt-4.1's. This is the clean axis-knockout signal:

> "Despite reasoning, when the underlying mechanic is obscured, the failure mode is axis-driven not prompt-driven."

Same Kaaus-quoted framing the meeting agreed to lead with. The data supports it.

## Caveats

- **n=2 is small.** Click-count differences look striking but error bars are large. Worth re-running with n=5 if budget allows.
- **OODA / OODA-F not run tonight** — meeting decision was to defer that sweep. Without OODA's orient-field instrumentation, we can't directly measure mechanic discovery (`d_turn` is None for everything because no orient text is captured outside OODA).
- **Wall-transfer never discovered.** The KA59 hidden mechanic remains undiscovered by either model, consistent with prior xAI/grok runs.
